搭建 VNC 服务

　　参考：linux ubuntu搭建不同用户的VNC

　　该文章详细介绍了，在服务器上搭建多用户的 vnc 服务需要的注意事项和相关命令。
并在 创建用户​ 中应用了这部分内容。

　　目前采用人工分配端口的方式，为用户分配 vnc 端口。可根据文章中提到的，不指定端口号启动 vnc 来自动分配（但这样每一次用户登陆，都新开启一个端口用于 vnc 登陆。可以修改 vnc 脚本来判断是否启动了当前用户的 vnc 服务，再决定要不要分配新的端口，目前未实现）。

创建用户

　　创建新用户需要：

  1. 创建 home 目录
  2. 创建用户及密码
  3. 添加到 docker 组
  4. 为用户配置 vnc 服务，分配 vnc 端口号

　　上述步骤，除手动分配 vnc 端口外，都通过脚本完成。

　　准备事项：1. 参考上一节 搭建 VNC 服务，安装对应的软件。2. 根据本节下述参考文章，创建对应的文件。

　　参考：ubuntu创建新用户并分配VNC远程桌面

　　该文章中介绍了添加用户所需要的文件、命令。需要根据其给出的脚本内容，在服务器上创建对应的文件。可以自行修改脚本路径。
然后按照使用方法，运行该脚本即可。

　　注意事项：

  1. 脚本中 vnc 启动脚本指定了端口号，添加完新用户后，需要修改该用户的 vnc 启动脚本中的vnc 端口号。
  2. 用户第一次登陆时，会运行 vnc 服务，提示输出 vnc 密码。可以在创建用户的时候，帮助其创建该密码。最好保持和ssh密码一致，防止遗忘。
